# Theory Notes

## Revision

The session began with a quick recap of the concepts covered on Day 2, including Linux architecture, file systems, and file permissions.

---

## Linux File System

We explored the Linux file system hierarchy and understood the purpose of important directories such as:

* `/`
* `/home`
* `/etc`
* `/bin`
* `/usr`
* `/var`
* `/tmp`

Understanding the file system is important for navigating and administering Linux systems.

---

## Linux File Permissions

We revised the three permission types:

* Read (r)
* Write (w)
* Execute (x)

along with the three permission groups:

* Owner
* Group
* Others

These permissions control access to files and directories and are fundamental to Linux security.

---

## Processes

We learned what processes are and how programs execute within the Linux operating system.

Topics included:

* Running processes
* Foreground and background processes
* Process identification (PID)
* Managing running processes

---

## Services

The session introduced Linux services (daemons) and explained how background services support operating system functionality.

Examples included:

* SSH
* Apache
* Cron

---

## Package Management

We learned how Linux software is installed, updated, and removed using package managers.

Topics included:

* Installing packages
* Updating packages
* Removing packages
* Package repositories

---

## Shell Scripting

A major portion of today's class focused on Bash scripting.

Topics covered included:

* Variables

## Comparison Operators

We learned various comparison operators used within shell scripts for decision-making.

* Numeric comparisons

## Case Statement

The `case` statement was introduced as an alternative to multiple `if-else` conditions, making scripts more readable and easier to maintain.

---

## Loop Statements

Different looping constructs were discussed for automating repetitive tasks.

Topics included:

* `for` loop
* `while` loop
* Loop control

---

## Common Beginner Mistakes

The instructor highlighted several common mistakes made while writing shell scripts, including:

* Incorrect spacing
* Missing quotation marks
* Incorrect variable usage
* Syntax errors
* File permission issues
* Missing shebang (`#!/bin/bash`)

Understanding these mistakes helps in writing cleaner and more reliable scripts.

---

## Cron Jobs

An introduction to Cron Jobs was provided.

We learned how scheduled tasks can be automated in Linux using cron, making it useful for repetitive administrative and security-related tasks.
