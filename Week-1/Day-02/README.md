# Theory Notes

## CIA Triad

The CIA Triad is the foundation of information security.

* **Confidentiality** ensures that sensitive information is accessible only to authorized users.
* **Integrity** ensures that information remains accurate and unaltered.
* **Availability** ensures that systems and data remain accessible whenever required.

Understanding these three principles is essential for designing secure systems.

## Zero-Day Exploits

A Zero-Day Exploit is an attack that targets a software vulnerability before the software vendor has released a patch or fix.

Key points discussed:

* Attackers exploit previously unknown vulnerabilities.
* No official security patch exists at the time of discovery.
* Zero-day attacks are considered highly dangerous due to the lack of immediate defenses.

---

## Cybersecurity Playbooks

Cybersecurity playbooks are predefined procedures that guide security teams during specific incidents.

Benefits include:

* Standardized incident response
* Faster decision-making
* Reduced response time
* Consistent handling of security incidents

---

## NIST Cybersecurity Framework

An introduction to the NIST Cybersecurity Framework was provided.

The framework consists of five core functions:

1. Identify
2. Protect
3. Detect
4. Respond
5. Recover

These functions help organizations effectively manage cybersecurity risks.

## Linux File Permissions

We learned how Linux controls access to files and directories through permissions.

Permission categories:

* Read (r)
* Write (w)
* Execute (x)

Permission groups:

* Owner
* Group
* Others

Understanding file permissions is critical for maintaining system security.

---

## SUID (Set User ID)

We were introduced to the SUID permission in Linux.

Key concepts:

* Allows a program to execute with the permissions of the file owner rather than the user running it.
* Commonly used for privileged system programs.
* Improperly configured SUID binaries can become privilege escalation vectors.
