# Day 08 - Vulnerability Scanning & Web Exploitation Basics

**Date:** 14 July 2026

---

## Objective

The objective of today's session was to introduce vulnerability assessment using Nessus and understand the fundamentals of web exploitation through practical labs on the ThunderCipher platform.

---

## Topics Covered

- Introduction to Nessus
- Nessus Installation & Configuration
- Basic Network Scanning
- Vulnerability Assessment
- Web2Shell Lab
- HTTP 403 Bypass Techniques
- Web Application Firewall (WAF) Rules
- Reverse Shell Concepts
- Reverse Shell Workflow

---

## Practical Activities

- Installed and configured Nessus Essentials.
- Performed a basic network vulnerability scan using Nessus.
- Explored scan results and identified detected services.
- Solved the **Web2Shell** room on the ThunderCipher platform.
- Learned common techniques used to bypass **HTTP 403 Forbidden** restrictions in authorized lab environments.
- Discussed how **Web Application Firewall (WAF)** rules help detect and block malicious requests.
- Understood how reverse shells are established between an attacker and a compromised machine.

---

## Key Learnings

- Understood the purpose of vulnerability assessment before exploitation.
- Learned how Nessus identifies security weaknesses within a network.
- Gained hands-on experience performing a basic network scan.
- Explored the Web2Shell exploitation workflow in a controlled lab environment.
- Learned the fundamentals of HTTP 403 bypass techniques.
- Understood the importance of properly configured WAF rules in protecting web applications.
- Developed a conceptual understanding of reverse shells and how they enable remote command execution after successful exploitation.

---

## Reflection

Today's session shifted the focus from reconnaissance and enumeration to vulnerability assessment and introductory web exploitation. Installing and using Nessus provided insight into how automated vulnerability scanners assist security professionals in identifying potential weaknesses. The Web2Shell lab reinforced the importance of proper web server configuration, access control, and defensive mechanisms such as Web Application Firewalls. Overall, the session provided a practical introduction to how vulnerabilities are identified and leveraged within an authorized penetration testing environment.
